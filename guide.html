Visitors

Getting started

  1. Use the public pages to listen on Spotify, learn about membership, see sample events, browse the store demonstration, or contact the team.
  2. Select the accessibility button marked “Aa” at the lower-left corner to adjust text, contrast, motion, link styling, font, and spacing.
  3. Create a free local demo account or open one of the included sample accounts to enter the member area.
  4. Use the account menu in the header to reach the dashboard, profile, messages, settings, or staff portal.

Member access

Accounts

Create a local test account

  1. Open Sign In and choose Create account.
  2. Enter a display name, username, email, and a test-only password.
  3. Confirm the age and policy checkboxes.
  4. The account is saved only in the current browser and opens at the Listener level.

Important limitation

This demonstration does not provide verified email, secure sessions, server-side password storage, password reset, multi-device sync, or administrator account approval. Those require a backend.

Member content

Library

  1. Search by title, description, or tag.
  2. Filter by content type or access level.
  3. Sign in with an account that meets the access label.
  4. Select Open resource to view or download the file.
  5. Select Save to place it on the dashboard.

Access levels

  • All members: any signed-in account.
  • Inner Circle: paid core membership and above.
  • Velvet Patron: highest member tier and staff.
  • Staff only: internal operations material.

Real private files must never be committed to a public GitHub Pages repository. Production files need authenticated storage and server-side access rules.

Invitations

Events and invites

  1. Open Events to see access, date, time, location, description, interest, and capacity.
  2. Sign in with an eligible account and select RSVP.
  3. Use Add to calendar to download the example calendar file.
  4. Open the invitation PDF where available.
  5. Review privacy, photography, participation, and accessibility expectations before attending.

Store

Separate purchases

The store demonstration supports products, quantities, a bag, totals, and a checkout handoff. It never asks for card data.

  1. Add sample books, games, tickets, or bundles to the bag.
  2. Review quantity and total.
  3. Select checkout to see the integration notice.
  4. Before launch, connect a secure hosted payment provider and confirm purchases through signed webhooks.
  5. Grant purchased downloads only after the backend verifies payment.

Member connection

Profiles and messages

The directory contains fictional sample profiles. Search by name or interest, open a profile, and start a conversation when messaging is allowed.

  • Do not include private contact information in a public or member-visible biography.
  • Respect “messages off” settings.
  • Use Block to stop local contact in the demo.
  • Use Report to create a test moderation record.
  • Staff should never read private messages without a clear, published moderation and legal basis.

Community care

Safety and moderation

Community rules should be public, specific, and enforced consistently. Members should know how reports are reviewed, what evidence is retained, how appeals work, and when an account may be suspended.

  • No harassment, coercion, threats, spam, impersonation, or unwanted contact.
  • No sharing another member’s identity, messages, photos, or event participation without permission.
  • No asking staff or members for passwords or payment card information.
  • Block and report controls should be available from profiles and messages.
  • Emergency and crisis matters should be directed to appropriate local services, not a routine website contact form.

Member control

Privacy settings

Open Settings to control directory visibility, messages, online status, location, interests, recommendations, release emails, and event emails. The Data section exports local demonstration information as JSON.

A production service also needs consent records, data retention rules, account deletion, verified export requests, staff access logs, and a process for privacy requests.

Inclusive use

Accessibility settings

The “Aa” button is available on every page. Settings can also be changed from the account Settings page.

  • Four text-size choices.
  • High-contrast mode.
  • Reduced motion.
  • Underlined links.
  • Simpler reading font.
  • Increased reading spacing.
  • Midnight and Twilight appearances.

Restricted operations

Staff uploads

  1. Open Sign In and choose the Studio Admin demo account.
  2. Open the account menu and select Staff portal.
  3. Enter a title, subtitle, type, access level, status, tags, and description.
  4. Choose a PDF, TXT, DOCX, EPUB, PNG, or JPG file up to 25 MB.
  5. Select Store and publish locally.
  6. Open Library with a matching account to test access.

Production upload rules

  • Validate extension, MIME type, file signature, and size on the server.
  • Rename files safely and prevent path traversal.
  • Scan uploads for malware.
  • Block script execution in storage directories.
  • Use private object storage with short-lived authorized download URLs.
  • Record uploader, access level, version, status, and audit history.
  • Use Public, Signed-in Member, Inner Circle, Velvet Patron, and Staff Only rules deliberately.

From prototype to service

Production launch checklist

  • Connect secure authentication and verified email.
  • Move profiles, messages, files, events, purchases, and contact submissions into a protected database.
  • Connect private object storage with server-side authorization.
  • Connect hosted subscription and one-time checkout.
  • Add rate limiting, CSRF protection, security headers, spam filtering, session management, and audit logs.
  • Write final privacy, terms, community, refund, moderation, and accessibility policies.
  • Replace sample products, events, documents, users, pricing, and contact information.
  • Test keyboard, screen reader, zoom, contrast, mobile layouts, forms, account recovery, and error states.

The included backend/schema.sql and docs/BACKEND-ROADMAP.md provide a starting technical plan.
